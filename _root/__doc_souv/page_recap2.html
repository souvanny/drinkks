Architecture Scalable

Approche: Design multi-tenant avec isolation physique/logique selon le segment client, auto-scaling horizontal.

Critère Portail Public (Multi-tenant) Solution Complète (Dédié)
Modèle d'isolation Logique (namespaces Kubernetes, schémas BDD) Physique (clusters séparés, instances BDD dédiées)
Base de données PostgreSQL avec schémas séparés PostgreSQL/MariaDB instance dédiée
Stockage Volumes partagés avec quotas Volumes dédiés avec performance garantie
Réseau Network policies Kubernetes VPC/VNet complètement isolé
Scalabilité Auto-scaling horizontal limité Auto-scaling horizontal/vertical complet
Monitoring Namespace dans cluster monitoring Stack monitoring dédié

Auto-scaling

Basé sur les métriques CPU/mémoire

  • HPA (Horizontal Pod Autoscaling)
  • Cluster Autoscaler pour les nodes
  • Scaling basé sur les métriques custom
  • Pré-scaling pour pics prévisibles

Partitionnement et réplication
